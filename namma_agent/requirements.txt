# Namma Agent (cloud-only) requirements.
#
# Install everything:        pip install -r namma_agent/requirements.txt
# Minimal (no voice/GUI):    install just the "CORE" + ONE provider section.
#
# The brain is an API call — you only need the SDK for the provider you use.

# ── CORE (required) ────────────────────────────────────────────────
fastapi
uvicorn[standard]        # ASGI server + websockets + http-tools
websockets               # comms: Discord Gateway bot + Slack Socket Mode (inbound, local)
pydantic
PyYAML

# ── PROVIDERS (install at least the one set in config.yaml) ────────
anthropic                # provider.type: anthropic   (default)
openai                   # provider.type: openai | openai_compat | ollama | lmstudio
google-genai             # provider.type: google

# Voice (TTS + STT) is 100% browser-native (Web Speech API) — no Python deps.

# ── DESKTOP WINDOW (optional — native window instead of browser) ───
pywebview                # `python -m namma_agent` opens a native window; falls back to browser
# pywebview needs an OS web engine to render in a native window:
#   Windows: WebView2 runtime (preinstalled on Win10/11; else the Evergreen
#            installer). The launcher forces the EdgeChromium engine so it's
#            smooth, not the laggy legacy MSHTML/IE fallback.
#   Linux:   GTK + WebKit2 — `sudo apt install python3-gi gir1.2-webkit2-4.1`
#            (or `pip install "pywebview[qt]"`). The launcher auto-bridges the
#            distro's system PyGObject into the venv, so usually no extra install.

# ── TOOL EXTRAS (optional — features degrade gracefully without these) ──
ddgs                     # web_search backend (renamed from duckduckgo-search; falls back to keyless HTML)
requests                 # render_diagram: hosted mermaid.ink rendering (primary, no local browser)
mermaid-cli              # render_diagram: LOCAL/offline mermaid → PNG fallback (drives headless Chromium via Playwright; run: playwright install chromium)
playwright               # controlled Chromium for real video playback + media_control + local mermaid render (run: playwright install chromium)
markitdown               # read_document: rich extraction (pdf/docx/pptx/xlsx/html/epub → markdown)
pypdf                    # read_document for .pdf (fallback)
python-docx              # read_document for .docx (fallback)
python-multipart         # document upload endpoint (multipart/form-data)
# convert_document: md → docx/pdf/pptx/html/txt/… . pandoc (a system binary, not pip —
#   https://pandoc.org/install) gives high-fidelity universal conversion; without it the
#   tool still does md/txt/html (stdlib) and docx (python-docx, above). 'markdown' (pip)
#   upgrades the html fallback when pandoc is absent.
markdown                 # convert_document: richer md → html fallback (optional)

# ── COGNEE ENHANCED MEMORY (optional — NO pip deps) ────────────────
# Cognee semantic + knowledge-graph memory runs FULLY CONTAINERIZED and is reached
# through the built-in MCP client (Namma launches cognee/cognee-mcp via `docker run`).
# It therefore adds NOTHING to this file — the only prerequisite is Docker.
# Setup: scripts/setup_cognee.ps1 (or .sh). Guide: docs/COGNEE.md.

# ── DEV ────────────────────────────────────────────────────────────
pytest
